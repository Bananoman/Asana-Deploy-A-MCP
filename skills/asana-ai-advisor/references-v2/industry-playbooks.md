# Industry Playbooks

These are starting patterns, not promises. Adapt them to the user's actual
project structure, access model, and source documents.

Bias toward narrow specialists.

---

## Executive Search / Recruiting

**Best first Teammates**
1. Search Brief Drafter — intake task -> structured search brief
2. Weekly Search Update Writer — recurring status task -> client-ready update draft
3. Interview Prep Packager — candidate stage change -> prep notes and open questions
4. Reference Question Builder — reference stage -> structured question set
5. Search Debrief Analyst — closeout task -> lessons learned and reusable notes

**Good AI Studio workflows / rules**
- assign follow-up owner based on search stage
- create Friday status task every week
- flag candidates with no activity for 7+ days

**Hybrid / external-agent patterns**
- candidate web sourcing
- relationship graph mapping
- compensation benchmarking from external sources
- CRM sync

**Avoid**
- promising fully autonomous pipeline creation if the team does not already use a stable project template

---

## Marketing / Content

**Best first Teammates**
1. Campaign Brief Writer — intake notes -> structured brief
2. Content Review Partner — draft task -> clarity, messaging, and CTA review
3. Launch Readiness Reviewer — pre-launch task -> risks, missing owners, and gaps
4. Performance Debrief Writer — campaign closeout -> learnings and next-step summary
5. Content Gap Synthesizer — recurring review task -> missing themes or asset gaps

**Good AI Studio workflows / rules**
- route content requests by type
- move to review when draft is complete
- remind when calendar gaps appear inside the next 2 weeks

**Hybrid / external-agent patterns**
- SEO research
- competitor content monitoring
- market research feeds

---

## Operations / Process

**Best first Teammates**
1. SOP Drafter — rough process notes -> structured SOP
2. Weekly Ops Briefer — recurring task -> cross-project digest
3. Risk Reviewer — project audit task -> risks and mitigation suggestions
4. Onboarding Plan Drafter — new hire task -> 30-60-90 draft
5. Vendor Review Packager — comparison task -> structured summary and decision criteria

**Good AI Studio workflows / rules**
- escalate overdue work after a fixed threshold
- create renewal review tasks before contract dates
- route inbound ops tasks by category

---

## Product / Engineering

**Best first Teammates**
1. Spec Reviewer — PRD or spec task -> missing details, dependencies, and questions
2. Sprint Risk Reviewer — recurring sprint task -> blockers and at-risk work summary
3. Bug Triage Summarizer — bug review task -> likely duplicates, severity notes, and next-step draft
4. Release Notes Drafter — closeout task -> user-facing summary
5. Retro Synthesizer — retro input task -> themes, wins, and follow-ups
6. Stakeholder Update Writer — milestone review task -> concise status summary

**Good AI Studio workflows / rules**
- move new bugs into triage
- assign P1 items to the on-call owner
- create QA follow-up when a dev task is completed

**Avoid**
- promising dependency creation, milestone creation, or bulk rescheduling through a Teammate

---

## Client Services / Agency

**Best first Teammates**
1. SOW Drafter — scope notes -> statement of work draft
2. Client Update Writer — recurring task -> polished weekly update draft
3. Kickoff Packager — kickoff task -> agenda, questions, and checklist
4. Scope Gap Reviewer — audit task -> likely scope creep and missing approvals
5. Debrief Analyst — closeout task -> lessons learned and reusable notes

**Good AI Studio workflows / rules**
- alert account manager when budget or timeline risk is triggered
- create invoice milestone review tasks before billing
- move deliverables to pending approval when complete

---

## Professional Services / Consulting

**Best first Teammates**
1. Engagement Brief Builder — intake notes -> structured engagement brief
2. Deliverable Reviewer — review task -> scope alignment and missing elements
3. Weekly Portfolio Synthesizer — recurring task -> cross-engagement summary
4. Knowledge Capture Drafter — closeout task -> reusable IP notes
5. Proposal Structure Writer — opportunity task -> proposal outline draft

---

## How to use these playbooks

1. Identify the team type from the workflow analysis.
2. Start with the most obvious Tier 1 candidate.
3. Check the current capability model before recommending a build.
4. Make access, sharing, and key resources explicit.
5. If the workspace spans multiple team types, pick at most 1 to 2 initial builds per team and sequence by impact and readiness.
